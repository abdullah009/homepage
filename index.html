k-mer Counting

A massive number of bioinformatics applications require counting of k-length substrings in genetically important long strings. A k-mer counter generates the frequencies of each k -length substring in genome sequences. Genome assembly, repeat detection, multiple sequence alignment, error detection,and many other related applications use a k-mer counter as a building block. Very fast and efficient algorithms are necessary to count k-mers in large data sets to be useful in such applications. We propose a novel trie-based algorithm for this k-mer counting problem.

We compare our devised algorithm KCMBT (k-mer Counter based on Multiple Burst Trees) with available all well-knownw that KCMBT is around 30% faster than the previous bestperforming algorithm KMC2 for human genome dataset. As another example, our algorithm is around six times faster than Jellyfish2. Overall, KCMBT is 20%-30% faster than KMC2 on five benchmark data sets when both the algorithms were run using multiple threads.

KCMBT is freely available on GitHub.

Record Linkage

Integrating data from multiple sources is a crucial and challenging problem. Even though there exist numerous algorithms for record linkage or deduplication, they suffer from either large time needs or restrictions on the number of datasets that they can integrate. We report efficient sequential and parallel algorithms for record linkage which handle any number of datasets and outperform previous algorithms. Our algorithms employ hierarchical clustering algorithms as the basis. A key idea that we use is radix sorting on certain attributes to eliminate identical records before any further processing. Another novel idea is to form a graph that links similar records and find the connected components.

Our sequential and parallel algorithms have been tested on a real dataset of 1,083,878 records and synthetic datasets ranging in size from 50,000 to 9,000,000 records. Our sequential algorithm runs at least two times faster, for any dataset, than the previous best-known algorithm, the two-phase algorithm using faster computation of the edit distance (TPA (FCED)). The speedups obtained by our parallel algorithm are almost linear. For example, we get a speedup of 7.5 with 8 cores (residing in a single node), 14.1 with 16 cores (residing in two nodes), and 26.4 with 32 cores (residing in four nodes).

The above algorithms use single linkage-based hierarchical clustering. This linkage comes with chaining problem. To resolve this issue, we propose efficient as well as reliable sequential and parallel complete linkage hierarchical clustering algorithms. We have tested our algorithms on datasets with millions of synthetic records. Experimental results show that our algorithms achieve nearly 100% accuracy. Parallel implementations achieve almost linear speedups.

Huge achievement in record linkage methods has pushed us to make them freely available for interested agencies. RLT-S tool (www.rlatools .com) implements our record linkage algorithm. This website also contains necessary information such as instructions, submission history, feedback, publications and some other sections to facilitate the usage of the tool. The source code is publicly available on GitHub.

Higher-order Spectra Analysis

Polyspectral estimation is a problem of great importance in the analysis of nonlinear time series that has applications in communications, sonar, radar, speech, and image processing, biomedical signal processing, geophysics, etc. For instance, higher order spectra have been used in unsupervised and supervised clustering in big data scenarios, as a test for Gaussianity, to suppress Gaussian noise, to characterize nonlinearities in any data, and so on. Any algorithm for computing the kth order spectra on a dataset of size n needs Ω(nk - 1) time since the output size will be Ω(nk - 1) as well. Given that we live in an era of big data, n could be very large. In this case, sequential algorithms might take unacceptable amounts of time. Thus it is essential to develop parallel algorithms. We offer generic sequential algorithms for computing higher order spectra (HOS) that are asymptotically faster than any published algorithm for HOS. In addition, we offer memory efficient algorithms. Also, we present optimal parallel implementations of these algorithms on parallel computing models such as the PRAM and the mesh. We also provide experimental results on our sequential and parallel algorithms. Our parallel implementation on a CPU cluster achieves good speedups.

Minimum Spanning Tree Problem

Finding minimum spanning trees (MST) in various types of networks is a well-studied problem in theory and practical applications. A number of efficient algorithms have been already developed for this problem. We present an efficient algorithm, namely Edge Pruned Minimum Spanning Tree (EPMST) algorithm, which combines ideas from randomized selection, Kruskal's algorithm and Prim's algorithm. The algorithm has a superior performance relative to the well-known algorithms especially when the graph is not very sparse.

Closest l-mers

With the advances in the next generation sequencing technology, huge amounts of data have been generated in biology. A bottleneck in dealing with such datasets lies in developing effective algorithms for extracting useful information from them. Algorithms for finding patterns in biological data pave the way for extracting crucial information from voluminous datasets. In this project we focus on a fundamental pattern, namely, the closest l-mers. Given a set of m biological strings S1, S2, ..., Sm and an integer l, the problem of interest is that of finding an l-mer from each string such that the distance among them is the least. This problem has many applications. An application of great importance is motif search. Algorithms for finding the closest l-mers have been used in solving the (l, d)- motif search problem. Here we have developed exact and approximate algorithms for this problem for the special case of m = 3. We consider the Euclidean distance metric if the sequences contain real numbers.
